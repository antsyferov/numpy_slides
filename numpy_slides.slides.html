NumPy¶

NumPy - это Python библиотека с открытым исходным кодом для работы с данными.

Возможности:

  • поддержка многомерных массивов (включая матрицы);
  • поддержка высокоуровневых математических функций, предназначенных для работы с многомерными массивами.

Написана с использованием языков C и Fortran

NumPy можно рассматривать как Open Source альтернативу MATLAB. MATLAB также интерпретируемый язык программирования. И NumPy и MATLAB используют библиотеку LAPACK (Linear Algebra Package) и позволяют пользователю писать быстрые программы, поскольку операции выполняются над массивами, а не над скалярами. MATLAB был специально разработан для линейной алгебры и имеет более компактный и понятный синтаксис для базовых операций линейной алгебры. NumPy же, в свою очередь, позволяет сделать многие вещи, которые невозможны в MATLAB, а также позволяет использовать все преимущества Python. Есть Python библиотеки, которые позволяют получить тот же функционал, который доступен в MATLAB.

NumPy¶

Области применения:

  • Обработка изображений;
  • Линейная алгебра;
  • Биология;
  • Машинное обучение;
  • и многие другие...

Изображение может быть представлено в виде массива, который хранит значения для RGB каналов или степени градации серого. Массивы можно использовать для хранения генома. NumPy используют для хранения и анализа данных в задачах машинного обучения. Также для хранения и анализа данных используют библиотеки pandas или GraphLab SFrame (GraphLab проприетарный, но SFrame Open Source).

Создание массивов с использованием модуля NumPy¶

Массивы в NumPy:

  • Эффективны;
  • Однородны;
  • Хранятся в памяти.

NumPy массивы состоят из 2 основных компонентов - буфер данных (то, что понимаем под массивом в C или Fortran - непрерывный блок памяти фиксированной длины) и информация о данных.

  1. Реализованы на C и Fortran
  2. Могут хранить данные только одного типа, но типы могут быть не только скалярными
  3. pandas тоже хранит данные в памяти, SFrame хранит данные в персистентом хранилище (например, на диске)
In [1]:
import numpy as np  # так принято импортировать NumPy
In [2]:
rows = 1000
columns = 2000

np_array = np.random.rand(rows, columns)  # ndarray со случайными значениями размерами 1000 * 2000
py_list = np_array.tolist()  # преобразует ndarray в Python list

%timeit [[elem ** 2 for elem in row] for row in py_list]  # "магическая" команда IPython
%timeit np_array ** 2  # broadcasting - возводим в квадрат каждый элемент массива

1 loop, best of 3: 200 ms per loop
100 loops, best of 3: 3.04 ms per loop

IPython содержит множество "магических" команд для различных задач. Они начинаются либо с одного знака % (для строки), либо с двух %% (для целой ячейки).

In [3]:
from IPython.display import Image  # для отображения изображений в IPython блокноте
def describe_array(np_array):
    print(repr(np_array))
    print('Тип элементов массива: {}'.format(np_array.dtype))
    print('Число измерений (осей): {}'.format(np_array.ndim))
    print('Размеры (форма): {}'.format(np_array.shape))  # кортеж (длиной `ndim`) длины массивов по каждой оси
    print('Число элементов массива: {}'.format(np_array.size))  # произведение `np.prod()` элементов `shape`
    print('Размер каждого элемента, байт: {}'.format(np_array.itemsize))

IPython.display - модуль с инструментами для отображения различных объектов в IPython. Используется для отображения Image, Audio, IFrame, YouTubeVideo, математических формул и многого другого.

repr - спросить, что о нем знают

dtype - data type

ndim - number of dimensions

shape может использоваться для изменения формы массива, если количество элементов останется неизменным (np_array.shape = <tuple с новыми размерами>)

Существует 5 основных механизмов создания массивов:

  1. Преобразование других Python структур (lists, tuples)
  2. Внутренние способы NumPy для создания массивов (arange, ones, zeros и др.)
  3. Чтение массивов с диска (как стандартные, так и кастомные форматы данных)
  4. Создание массивов из "сырых" байтов, с использованием строк или буфферов
  5. Использование специальных функций (из модуля random)

Операции над массивами выполняются поэлементно, то есть оператор "*" не вернет произведение матриц. Для получения произведения матриц есть метод dot. В Python 3.5+ есть оператор "@" (метод matmul добавлен в NumPy 1.10) для умножения матриц, который не всегда возвращает тот же результат, что и dot, поэтому здесь следует быть осторожным. В NumPy также существует унаследованный класс matrix, операции над его объектами выполняются по правилам линейной алгебры. Объекты matrix могут быть только двумерными. Для поэлементного перемножения объектов matrix нужно использовать multiply.

Функция diag позволяет создать матрицу или получить диагональ матрицы.

Может принимать два параметра - array_like объект v и целое число k - номер диагонали. По умолчанию k=0 - главная диагональ, если k>0 - выше главной и наоборот.

Если параметр v одномерный массив, то в результате мы получим матрицу с массивом v на k-той диагонали.

Если параметр v двумерный массив, то в результате мы получим массив, содержащий копию k-той диагонали матрицы v.

Массивы в NumPy поддерживают многомерные индексы. Так, в Python мы бы получили TypeError.

В Python здесь мы тоже получим TypeError. NumPy также поддерживает второй вариант индексирования, но он будет работать медленнее, так как создается временный массив при первом индексировании.

Broadcasting - позволяет нам работать со входными данными разных размеров. Здесь мы присваиваем 0.5 каждому элементу второй строки.

Есть 4 правила broadcasting:

  1. All input arrays with ndim smaller than the input array of largest ndim, have 1’s prepended to their shapes.
  2. The size in each dimension of the output shape is the maximum of all the input sizes in that dimension.
  3. An input can be used in the calculation if its size in a particular dimension either matches the output size in that dimension, or has value exactly 1.
  4. If an input has a dimension size of 1 in its shape, the first data entry in that dimension will be used for all calculations along that dimension. In other words, the stepping machinery of the ufunc will simply not step along that dimension (the stride will be 0 for that dimension).
